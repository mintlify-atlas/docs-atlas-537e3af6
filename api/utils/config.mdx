## Overview

The configuration module provides utilities for managing command-line arguments, logging, and configuration parameters throughout OS-Copilot. It uses a singleton pattern to ensure consistent configuration access across the application.

## Classes

### Config

A singleton class for storing and accessing configuration parameters.

```python
class Config:
    """Singleton class for storing and accessing configuration parameters."""
```

#### Methods

<ParamField path="initialize" type="classmethod">
  Initializes the Config instance with command-line arguments.
  
  **Parameters:**
  - `args` (argparse.Namespace): The parsed command-line arguments
  
  ```python
  Config.initialize(args)
  ```
</ParamField>

<ParamField path="get_parameter" type="classmethod">
  Retrieves a parameter value by key.
  
  **Parameters:**
  - `key` (str): The key of the parameter to retrieve
  
  **Returns:**
  - Any: The value of the parameter if found, otherwise None
  
  ```python
  working_dir = Config.get_parameter('working_dir')
  ```
</ParamField>

## Functions

### setup_config()

Sets up configuration parameters based on command-line arguments and initializes logging.

```python
def setup_config() -> argparse.Namespace
```

**Returns:**
- `argparse.Namespace`: The parsed command-line arguments

**Behavior:**
- Creates an argument parser with predefined configuration options
- Parses command-line arguments (or uses defaults in test environments)
- Initializes the Config singleton
- Creates logging directory if it doesn't exist
- Configures logging with file output

### setup_pre_run()

Sets up pre-run tasks and formats the task description for logging.

```python
def setup_pre_run(args: argparse.Namespace) -> str
```

**Parameters:**
<ParamField path="args" type="argparse.Namespace" required>
  The parsed command-line arguments from setup_config()
</ParamField>

**Returns:**
- `str`: A formatted string containing task information

**Behavior:**
- Formats task query and optional file paths
- Prints task information to console
- Logs task information
- Returns formatted task string

## Usage Example

```python
from oscopilot.utils.config import setup_config, setup_pre_run, Config

# Initialize configuration
args = setup_config()

# Setup task execution
task = setup_pre_run(args)

# Access configuration parameters
working_dir = Config.get_parameter('working_dir')
max_iterations = Config.get_parameter('max_repair_iterations')

print(f"Working directory: {working_dir}")
print(f"Max repair iterations: {max_iterations}")
```

## Environment Variables

The configuration module loads environment variables from a `.env` file using `python-dotenv`. The `.env` file should be placed in the project root directory.

## Logging Configuration

The module automatically configures logging with:
- **File output**: Logs written to `{logging_filedir}/{logging_filename}`
- **Log level**: INFO
- **Format**: `[{logging_prefix}] {timestamp} - {level} - {message}`
- **Directory creation**: Automatically creates the logging directory if it doesn't exist

## Notes

- The Config class uses the singleton pattern to ensure only one instance exists
- In test environments (when pytest is imported), default values are used instead of parsing command-line arguments
- The logging prefix is randomly generated by default for unique identification across runs
- Configuration parameters persist throughout the application lifecycle once initialized
