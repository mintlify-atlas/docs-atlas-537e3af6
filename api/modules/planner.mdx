## Overview

The `FridayPlanner` class is responsible for decomposing complex tasks into manageable subtasks, replanning tasks based on new insights or failures, and managing the execution order of tasks.

The planner uses a combination of tool descriptions, environmental state, and language learning models to dynamically create and adjust plans for task execution. It maintains a tool graph to manage task dependencies and execution order, ensuring that tasks are executed in a sequence that respects their interdependencies.

### decompose_task()

Decomposes a complex task into manageable subtasks and updates the tool graph.

```python
@api_exception_mechanism(max_retries=3)
def decompose_task(self, task, tool_description_pair)
```

**Location:** `oscopilot/modules/planner/friday_planner.py:35`

This method takes a high-level task and a tool-description pair, and utilizes the environment's current state to format and send a decomposition request to the language learning model. It then parses the response to construct and update the tool graph with the decomposed subtasks, followed by a topological sort to determine the execution order.

<ParamField path="task" type="str" required>
  The complex task to be decomposed into subtasks
</ParamField>

<ParamField path="tool_description_pair" type="dict" required>
  A dictionary mapping tool names to their descriptions
</ParamField>

**Side Effects:**
- Updates the tool graph with the decomposed subtasks
- Reorders tools based on dependencies through topological sorting
- Modifies `tool_node`, `tool_graph`, and `sub_task_list` attributes

**Example:**

```python
planner = FridayPlanner(prompt_config)
tool_descriptions = {
    "FileReader": "Reads file contents from disk",
    "DataProcessor": "Processes and transforms data"
}
planner.decompose_task(
    task="Analyze the sales data from Q1 report",
    tool_description_pair=tool_descriptions
)
```

### topological_sort()

Generates a topological sort of the tool graph to determine the execution order.

```python
def topological_sort(self)
```

**Location:** `oscopilot/modules/planner/friday_planner.py:230`

This method applies a topological sorting algorithm to the current tool graph, considering the status of each tool. It aims to identify an order in which tools can be executed based on their dependencies, ensuring that all prerequisites are met before a tool is executed. The sorting algorithm accounts for tools that have not yet been executed to avoid cycles and ensure a valid execution order.

**Returns:**
- `None` if topological sort succeeds
- `str`: "Cycle detected in the graph, topological sort not possible." if a cycle is found

**Side Effects:**
- Populates `sub_task_list` with the sorted order of tools to be executed
- Only includes tools that haven't been executed yet

**Example:**

```python
planner.topological_sort()
if planner.sub_task_list:
    print(f"Execution order: {planner.sub_task_list}")
```

## Usage Example

```python
from oscopilot.modules.planner import FridayPlanner

# Initialize the planner
prompt_config = {
    '_SYSTEM_TASK_DECOMPOSE_PROMPT': '...',
    '_USER_TASK_DECOMPOSE_PROMPT': '...',
    '_SYSTEM_TASK_REPLAN_PROMPT': '...',
    '_USER_TASK_REPLAN_PROMPT': '...'
}
planner = FridayPlanner(prompt=prompt_config)

# Decompose a complex task
task = "Create a data analysis report from CSV files"
tools = {
    "ReadCSV": "Reads CSV files from disk",
    "AnalyzeData": "Performs statistical analysis",
    "GenerateReport": "Creates formatted reports"
}
planner.decompose_task(task, tools)

# Get execution order
for subtask in planner.sub_task_list:
    print(f"Execute: {subtask}")
    # Execute the subtask
    planner.update_tool(subtask, return_val="result", status=True)

# Replan if needed
if error_occurred:
    planner.replan_task(
        reasoning="File format was different than expected",
        current_task="ReadCSV",
        relevant_tool_description_pair={"ReadExcel": "Reads Excel files"}
    )
```
