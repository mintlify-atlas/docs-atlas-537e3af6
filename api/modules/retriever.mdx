## Overview

The `FridayRetriever` class is responsible for retrieving and managing available tools from the tool library. It extends the `BaseModule` class, focusing on the retrieval of tools based on specific prompts or queries.

The retriever interacts with a language learning model (LLM) and utilizes the execution environment and tool library to fulfill its responsibilities, including tool discovery, filtering, and metadata retrieval.

### retrieve_tool_name()

Retrieves a list of tool names relevant to the specified task.


**Location:** `oscopilot/modules/retriever/vector_retriever.py:33`

### tool_code_filter()

Filters and retrieves the code for a tool relevant to the specified task.

```python
def tool_code_filter(self, tool_code_pair, task)
```

**Location:** `oscopilot/modules/retriever/vector_retriever.py:50`

This method formats a message for filtering tool codes based on a given task, sends the message to the LLM for processing, and retrieves the filtered tool's code. If a tool name is successfully identified, its corresponding code is fetched from the tool library.

<ParamField path="tool_code_pair" type="dict" required>
  A dictionary mapping tool names to their codes
</ParamField>

<ParamField path="task" type="str" required>
  The task based on which the tool code needs to be filtered
</ParamField>

**Returns:**
- `str`: The code of the tool relevant to the specified task, or an empty string if no relevant tool is found

**Example:**

```python
tool_codes = {
    "PDFParser": "class PDFParser:...",
    "ImageProcessor": "class ImageProcessor:...",
    "TextExtractor": "class TextExtractor:..."
}

filtered_code = retriever.tool_code_filter(
    tool_code_pair=tool_codes,
    task="Extract text content from PDF documents"
)
print(filtered_code)
# Output: "class PDFParser:..." (most relevant tool for the task)
```

## Usage Example

```python
from oscopilot.modules.retriever import FridayRetriever

# Initialize the retriever
prompt_config = {
    '_SYSTEM_ACTION_CODE_FILTER_PROMPT': '...',
    '_USER_ACTION_CODE_FILTER_PROMPT': '...'
}
retriever = FridayRetriever(
    prompt=prompt_config,
    tool_manager=tool_manager_instance
)

# Retrieve relevant tools for a task
task = "Process and analyze customer feedback from CSV files"
relevant_tools = retriever.retrieve_tool_name(task, k=5)

# Get descriptions for the tools
tool_descriptions = retriever.retrieve_tool_description_pair(relevant_tools)
for tool_name, description in tool_descriptions.items():
    print(f"{tool_name}: {description}")

# Get the actual code for selected tools
tool_codes = retriever.retrieve_tool_code_pair(relevant_tools)

# Filter to find the most relevant tool
best_tool_code = retriever.tool_code_filter(
    tool_code_pair=tool_codes,
    task=task
)

print(f"Selected tool code:\n{best_tool_code}")
```

## Integration with Tool Manager

The `FridayRetriever` relies heavily on the `tool_manager` instance for accessing the tool library:

- **Vector Search**: Uses vector embeddings to find semantically similar tools
- **Metadata Access**: Retrieves tool descriptions, code, and parameters
- **Tool Management**: Supports adding and deleting tools from the library
- **LLM-Assisted Filtering**: Leverages language models to select the most appropriate tool for a given task

The retriever acts as an intelligent interface between the planning/execution modules and the tool library, ensuring that the most relevant tools are selected for each task.
