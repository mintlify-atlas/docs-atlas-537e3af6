## Overview

The `FridayExecutor` class is a module within the system responsible for executing tools based on prompts and maintaining the tool library. It focuses on the practical execution of tools determined by the system.

The executor utilizes a language learning model (LLM) in conjunction with an execution environment and a tool library to carry out tools. It handles code generation, execution, error analysis, tool repair, and tool storage operations.

### analysis_tool()

Analyzes the execution outcome of a tool to determine the nature of any errors.

```python

```

**Location:** `oscopilot/modules/executor/friday_executor.py:234`

This method evaluates the execution state of a tool, specifically looking for errors. Based on the analysis, it determines whether the error is environmental and requires new operations (handled by the planning module) or is amendable via the `repair_tool` method.

<ParamField path="code" type="str" required>

</ParamField>

<ParamField path="task_description" type="str" required>

</ParamField>

<ParamField path="state" type="object" required>
  The state object containing the result of the tool's execution, including any errors
</ParamField>

**Returns:**
- `tuple`: A tuple containing:
  - `reasoning (str)`: The analysis's reasoning regarding the nature of the error
  - `type (str)`: The type of error identified ('environmental' for new operations, 'amendable' for corrections)

**Example:**

```python

    code=failed_code,
    task_description="Write data to file",
    state=error_state

    planner.replan_task(...)
elif error_type == "amendable":
    # Can be fixed by repair_tool
    executor.repair_tool(...)
```

## Usage Example

```python
from oscopilot.modules.executor import FridayExecutor

# Initialize the executor
prompt_config = {
    '_SYSTEM_PYTHON_SKILL_AND_INVOKE_GENERATE_PROMPT': '...',
    '_USER_PYTHON_SKILL_AND_INVOKE_GENERATE_PROMPT': '...',
    '_SYSTEM_TASK_JUDGE_PROMPT': '...',
    '_USER_TASK_JUDGE_PROMPT': '...',
    # ... other prompts
}
executor = FridayExecutor(
    prompt=prompt_config,
    tool_manager=tool_manager_instance,
    max_iter=3
)

# Generate and execute a tool
code, invoke = executor.generate_tool(
    task_name="process_data",
    task_description="Process CSV data and calculate statistics",
    tool_type="Python",
    pre_tasks_info={"read_file": {"description": "Read CSV", "return_val": "data"}},
    relevant_code={}
)

# Execute the generated tool
state = executor.execute_tool(code, invoke, "Python")

# Judge the execution
reasoning, success, score = executor.judge_tool(
    code=code,
    task_description="Process CSV data and calculate statistics",
    state=state,
    next_action="generate_report"
)

if not success:
    # Analyze the error
    error_reasoning, error_type = executor.analysis_tool(
        code=code,
        task_description="Process CSV data and calculate statistics",
        state=state
    )
    
    if error_type == "amendable":
        # Repair the tool
        new_code, new_invoke = executor.repair_tool(
            current_code=code,
            task_description="Process CSV data and calculate statistics",
            tool_type="Python",
            state=state,
            critique=reasoning,
            pre_tasks_info={}
        )
        # Try again with repaired code
        state = executor.execute_tool(new_code, new_invoke, "Python")

if success and score > 0.8:
    # Store successful tool for future use
    executor.store_tool("process_data", code)
```

## Execution Lifecycle

The typical execution lifecycle managed by `FridayExecutor`:

1. **Generation**: `generate_tool()` creates executable code based on task requirements
2. **Execution**: `execute_tool()` runs the generated code in the environment
3. **Judgment**: `judge_tool()` evaluates whether the execution was successful
4. **Analysis**: If failed, `analysis_tool()` determines the error type
5. **Repair**: If amendable, `repair_tool()` fixes the code (up to `max_iter` times)
6. **Storage**: If successful, `store_tool()` saves the tool for future reuse

This lifecycle ensures robust tool execution with automatic error handling and self-improvement capabilities.
