## Prerequisites

Before starting, make sure you have:

- Completed the [installation](/installation) steps
- Activated your conda/virtual environment
- Configured your `.env` file with API keys

## Your First Agent in 5 Minutes

Let's run your first OS-Copilot agent using the included quick start script.

<Steps>
  <Step title="Navigate to the Project Directory">
    Open your terminal and navigate to the OS-Copilot directory:
    
    ```bash
    cd OS-Copilot
    ```
  </Step>
  
  <Step title="Activate Your Environment">
    Activate the conda environment you created during installation:
    
    ```bash
    conda activate oscopilot_env
    ```
  </Step>
  
  <Step title="Run the Quick Start Script">
    Execute the quick start script with the default task:
    
    ```bash
    python quick_start.py
    ```
    
    This will run a demonstration task that searches for text files containing the word "agent" and organizes them into a new folder.
  </Step>
</Steps>

## Understanding the Quick Start Script

Let's examine what happens when you run `quick_start.py`:

```python quick_start.py
from oscopilot import FridayAgent
from oscopilot import ToolManager
from oscopilot import FridayExecutor, FridayPlanner, FridayRetriever
from oscopilot.utils import setup_config, setup_pre_run

# Load configuration from command line and .env file
args = setup_config()

# Set default task if none provided
if not args.query:
    args.query = "Copy any text file located in the working_dir/document directory that contains the word 'agent' to a new folder named 'agents' "

# Prepare the task for execution
task = setup_pre_run(args)

# Initialize the FRIDAY agent with all modules
agent = FridayAgent(
    FridayPlanner,    # Decomposes tasks into sub-tasks
    FridayRetriever,  # Finds relevant tools from repository
    FridayExecutor,   # Executes code and handles errors
    ToolManager,      # Manages tool storage and retrieval
    config=args
)

# Run the task
agent.run(task=task)
```

## What Happens During Execution

When you run a task, FRIDAY follows this workflow:

<Steps>
  <Step title="Task Planning">
    The **Planner** analyzes your request and breaks it down into sub-tasks:
    
    ```text
    The task list obtained after planning is:
    ['search_text_files', 'filter_by_keyword', 'create_folder', 'copy_files']
    ```
  </Step>
  
  <Step title="Tool Retrieval">
    The **Retriever** searches for relevant tools in the repository that can help execute each sub-task.
  </Step>
  
  <Step title="Code Generation & Execution">
    The **Executor** generates code for each sub-task and executes it:
    
    ```python
    # Example generated code for file search
    import os
    import glob
    
    def search_text_files(directory):
        files = glob.glob(os.path.join(directory, "*.txt"))
        return files
    ```
  </Step>
  
  <Step title="Error Handling & Refinement">
    If errors occur, FRIDAY automatically attempts to fix them or replans the approach:
    
    ```
    current amend times: 1
    Attempting to repair code based on error feedback...
    ```
  </Step>
  
  <Step title="Task Completion">
    Once successful, the results are displayed:
    
    ```
    The execution of the current sub task has been successfully completed.
    ✓ Task completed successfully
    ```
  </Step>
</Steps>

## Advanced Usage

### Custom Configuration

You can customize the agent's behavior by modifying configuration parameters:

```python
from oscopilot import FridayAgent
from oscopilot import ToolManager
from oscopilot import FridayExecutor, FridayPlanner, FridayRetriever
from oscopilot.utils import setup_config

args = setup_config()

# Customize settings
args.max_repair_iterations = 5  # Max attempts to fix errors
args.score = 8.0  # Minimum score to store tools

agent = FridayAgent(
    FridayPlanner,
    FridayRetriever,
    FridayExecutor,
    ToolManager,
    config=args
)

# Run with custom task
agent.run(task="Your custom task here")
```

## Understanding Agent Modules

The FridayAgent consists of three core modules that work together:

### FridayPlanner

Decomposes high-level tasks into executable sub-tasks:

- Analyzes task complexity
- Identifies required capabilities
- Creates an execution plan
- Handles replanning when needed

### FridayRetriever

Finds relevant tools and code from the repository:

- Uses semantic search to match tasks with tools
- Retrieves code examples from previous executions
- Provides context for code generation

### FridayExecutor

Generates and executes code to accomplish tasks:

- Supports Python, Shell, and AppleScript
- Handles execution errors automatically
- Attempts repairs up to `max_repair_iterations` times
- Stores successful code in the tool repository

<Info>
These modules operate in a continuous feedback loop: **Plan → Retrieve → Execute → Judge → Repair/Replan**
</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Task fails immediately">
    **Issue**: The task fails without attempting execution
    
    **Solution**:
    - Check your `.env` file for correct API keys
    - Verify your OpenAI API has sufficient credits
    - Ensure the task is clearly defined and specific
  </Accordion>
  
  <Accordion title="Agent gets stuck in repair loop">
    **Issue**: The agent repeatedly tries to fix the same error
    
    **Solution**:
    - The task may be too complex or ambiguous
    - Try breaking it into smaller, more specific tasks
    - Check if required files/directories exist
  </Accordion>
  
  <Accordion title="Import errors when running">
    **Issue**: `ModuleNotFoundError` or import errors
    
    **Solution**:
    ```bash
    conda activate oscopilot_env
    pip install -e . --upgrade
    ```
  </Accordion>
  
  <Accordion title="Permission denied errors">
    **Issue**: Cannot access or modify files
    
    **Solution**:
    - Ensure you have write permissions in the working directory
    - Run from a directory where you have full access
    - Check file/folder permissions with `ls -la`
  </Accordion>
</AccordionGroup>

## Important Notes

<Warning>
**Single-Round Conversation**: FRIDAY currently supports only single-round conversations. Each execution is independent and doesn't maintain context between runs.
</Warning>

<Warning>
**System Access**: OS-Copilot can execute code and modify your system. Always review tasks carefully and run in a safe environment. The developers are not liable for any data loss or system changes.
</Warning>
